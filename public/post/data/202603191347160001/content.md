# Primitive Type Variable
### c언어
일반적으로 c계열 언어에서 발전한 java, C#, kotlin 등에서 사용되는 개념이다. c의 경우 변수를 선언했을 때 개발자는 그 변수에 대한 포인터를 얻어 해당 공간의 값을 제어할 수 있었다(c의 원시 타입을 위한 철학도 업적이기에 나중에 알아보도록 하자). 공간에서 값을 직접 지정할 수 있기 때문에 원시 타입이라는 단어가 struct나 pair로 만든 커스텀 변수가 아닌 int, char 등 최소 단위의 변수 예약어를 의미했다.  

### OOP 언어
그러나 java, C#이 나오면서 원시 타입의 의미는 다르게 사용되기 시작했다. 변수의 타입이 아닌 변수를 다루는 방식으로 구분이 되었다.
reference라는 개념이 OOP와 함께 언어 수준에서 적용되면서 변수를 만들거나 복사할 때 실제 값을 만들거나 복사하는가를 여부로 값 변수와 참조 변수라는 이분법적 형태로 나뉘게 되었다. 값을 다루기 위해서 사용할 수 있는 타입을 원시 타입이라고 지정했고 값 변수를 만들기 위해선 원시 타입으로 만들어야 했다. 이외는 모두 참조 변수로 참조값만 저장한다.

### Java script
위의 개념으로 사용되는 원시 타입이라는 의미가 JS에서 똑같이 사용된다. 그러나 동작 원리가 아예 다르다.
일반적인 언어에선 값 변수를 변경한다면 식별자가 가리키는 공간 내부를 수정한다.
그러나 JS의 경우 변경한 값을 위한 새로운 공간을 만든다는 점이 특징이다.
먼저 새로운 값이 담겨있는 공간이 존재하는지 확인한다. 존재하면 그 공간 자체를 식별자에 연결한 후 마무리 짓는다.
만약 공간이 없다면 값을 만든 뒤 식별자가 가리키는 공간에 연결한다.
이전 값 공간에 대한 연결된 식별자가 있다면 그대로 해당 영역에 납두지만, 없을 경우 GC가 해당 공간을 초기화해버린다.
데이터가 생성되고 사라질 때까지 변화하지 않으니 데이터에 불변성(Immutablility) 개념이 적용되었다고 한다.

## Java script 원시 타입
들어가기에 앞서 JS의 원시 타입이 뭐가 있는지 살펴보자. JS의 원시 타입은 공식문서에 정확하게 명시되어 있다.

- number : 64bit 부동소수점
- bigInt : 매우 큰 정수
- string : 문자열
- boolean : 논리
- null : 비어있음
- undefined : 값을 지정하지 않음
- symbol : 고유한 식별자

## Java script 값 변수 시스템의 장단점
### 장점
1. 중복 데이터에 대한 메모리 저장 공간 확보
2. 동일 데이터인지 확인할 때 데이터 크기에 상관없이 식별자에 들어있는 참조값만 확인하여 빠름

### 단점
1. 수정에 대한 비용이 생성보다 높음
2. 수정할 때 GC 과부하 가능성

단점에 대해 예시를 하나 들어보면 문자열 500MB를 어쩔 수 없이 선언했는데 이를 지속적으로 수정해야 하는 케이스다. 수정할 때 마다 500MB가 만들어지고 사라지고 하면 GC가 터질 것이다. 이에 대한 여러가지 방법론이 있다.
1. 실시간 수정 막기 : 수정할 때마다 수정하면 문자열이 계속 새로 생기고 지워진다. 그러기에 버퍼 개념을 두어 일정 기간 마다 수정하도록 한다. 이래봤자 500MB 새로 생기는 건 같다.
2. 참조형 객체로 문자열 500MB 공간을 확보해서 사용한다. 다만 단점은 동일 데이터인지 확인할 때 새로운 로직을 둬야 한다. 그러나 이게 훨씬 합리적이기 때문에 이 방식을 이용해야 한다.
