# Heap이란?

개념 자체는 쌓아 올린다는 의미이다. 이 개념엔 한 가지 규칙이 있다. 쌓아 올리기 위해 절대 무너지면 안 된다. 이번 포스팅에서 다룰 Heap Tree는 무너지지 않게 노드들을 완전 이진 트리로 쌓아 올린다. 또한 Memory 영역에서 사용되는 Heap도 무너지지 않게 메모리 영역에 크기에 따라 규칙적으로 쌓아 올린다. 모양과 목적은 다르지만 Heap 개념은 결국 무너지지 않게 쌓아 올린다는 것을 의미한다는 것을 유념해야 할 것이다.

# Heap의 구조적 특징

Heap을 알기 위해서는 이진 트리에 대해서 알아야 한다. 그리고 데이터가 쌓일수록 탐색과 정렬에서 더 빛을 내는 완전 이진 트리를 알아야 한다. 이것도 부족해서 Heap이라는 자료구조가 왜 나왔는지 파악해야 한다. 이진 트리와 완전 이진 트리는 형태를 살펴보고 그 특징이 어떨지 간단하게만 정리하고 넘어가겠다.

## 이진 트리
단순하게 자식이 최대 2개면 된다.

## 완전 이진 트리
위에서 아래로, 옆에서 반대편으로 빈틈없이 채워진 이진 트리이다.

## Heap
최대값 혹은 최소값이 루트인 완전 이진트리이다.

# Heap Tree 사용 예제
## Prioriry Queue
루트 빼버린 다음에 최대값 아무거나 놓고 밑으로 빼버리면 됨

## Heap sort
루트 빼버린 다음에 최대값 아무거나 놓고 밑으로 빼버리는걸 N번 하면 됨

## Dijkstra Algorithm
매 탐색마다 Priority Queue에서 빼버리면 됨

# Heap Operation
루트가 최대값 기준으로 설명
## push
맨 밑에 넣고 자기보다 큰 노드 만날 때까지 올려보내면 됨

## pop
root 빼버리고 맨 아래 노드 하나를 맨 위로 올린 다음 자기보다 작은 노드 만날 때 까지 내려보내면 됨
