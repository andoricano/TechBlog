# Abstract Data Type(ADT)

다익스트라 알고리즘을 찾아보는데 우선 순위 큐가 사용되었다. 우선 순위 큐를 오랜만에 접하게 되어서 다시 한 번 찾아보는데 추상 자료형이라는 단어가 나왔다. 의미는 와닿으나 생소한 단어였기 때문에 찾아보니 소프트웨어 개발에서 주요한 개념을 가지고 있다고 생각하여 정리해놓고자 한다.

## 나오게 된 배경
- 1960년대 후반의 소프트웨어 위기에 대응하기 위함
- 모듈화의 방법론 필요성
- 데이터 관리의 필요성

## 그래서 ADT란?
자료 자체의 형태와 그 자료에 관계된 연산들을 추상적으로만 정의한 것. 간단하게 말하자면 "이 데이터 형태는 A이고 이것 저것 할 수 있어야 해요"라고 말하면 ADT인 것이고 "그것을 만들기 위해 무엇을 사용해야 해요" 라고 하면 자료구조가 되는 것이다.  
우서 순위 큐때문에 ADT를 정리하기로 했으니 우선 순위 큐를 예시로 들겠다.

## 우선 순위 큐와 ADT
* 우선 순위 큐의 개념 : 큐에 항목과 우선순위의 쌍(자료 자체의 형태)을 넣어 가장 우선적인 것을 먼저(관계된 연산1) 내보낸다. 같을 경우 선입선출(관계된 연산2).

* 우리는 이를 위해 배열로 빠르게 만들든가, 연결 리스트로 만들어서 삽입 및 삭제를 편하게 구현하든가, Heap을 통해 최적의 성능을 기대하든가 내 마음대로 구현할 수 있다. 이로써 자료구조가 되는 것이다.
