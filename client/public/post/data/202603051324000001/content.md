# 1. 목적
JS는 SW 개발자에게 필수적인 역량이 되었다고 해도 과언이 아니다. 그러나 태생부터, 현재 생태계까지 발전 과정이 
복잡하기 때문에 이러한 부분은 좀 나중에 파악하도록 하고, 해당 문서는 JS가 무엇인지, 어떻게 실행되고 사용되는지에 대해 간단하게 정리한다.

# 2. Java Script란?
WEB에서 HTML과 CSS만으로 부족해서 출범한 프로그래밍 언어이다. 즉 WEB을 위한 언어라고 보면 된다. WEB 브라우저의 경우 JS를 해석하는 엔진을 갖고 있어서 JS 코드를 마주하면 그대로 실행할 수 있다. 보통 HTML과 CSS가 처리할 수 없는 interface 적인 요소를 해결한다. 그리고 WEB 뿐만 아니라 Server, Game Script 등 다양한 곳에서 사용되기 시작되었다. 이 언어의 플랫폼까지 생겨나 Node JS 생태계까지 만들어진다.

## 2.1 프로그래밍 언어로써의 특징
JS의 특징 중 가장 중요한 점은 타입 관련하여 에러를 잡지 않는다. a = 1로 선언했으나 언젠가 만날 a는 "에이" 일 수도 있다는 것이다. 이를 위해 Type Script까지 개발되어 지금 널리 사용 중이다.
그리고 class 문법이 갑자기 생긴 이력이 있다. 이는 프로토타입이라는 기반 위에 class라는 문법을 덮어 씌운 거나 마찬가지라고 한다. 프로토타입의 뜻은 시제품 혹은 원형으로 보면 된다. 아직 확연하게 그 개념이 떠오르지 않는다.
여기서 전혀 상관없는 언어인 Java의 Object와 비교하면 방향성을 잡을 수 있다고 본다.

| 구분     | Java의 Object (클래스 기반)          | JS의 Prototype (프로토타입 기반)                |
| :------- | :----------------------------------- | :---------------------------------------------- |
| **정체** | 모든 클래스가 상속받는 최상위 설계도 | 모든 객체가 연결된 최상위 실체(객체)            |
| **시점** | 코드를 실행하기 전(컴파일)에 결정됨  | 코드가 돌아가는 중(실행 중)에도 바꿀 수 있음    |
| **방식** | "나는 Object의 자식이다"라고 명시함  | 부모 객체의 주소를 참조만 하고 필요할 때만 접근 |

마치 자취생이 부모님께 전수받은 실력으로 반찬은(Java) 안하고 본가가서 냉장고에서 반찬만 빼오는거(JS)라 생각하면 된다.

## 2.2 Java Script 엔진
위에서 엔진이라는 말이 나왔다. JS의 엔진은 뭘까? 보통 브라우저마다 각각의 이름으로 JS를 실행시키는 프로그램이 있는데 이게 엔진이다. 여기서 가장 유명한게 구글에서 개발한 크롬의 V8 엔진이라고 하는데 이거를 또 Node.js에서 떼어다가 사용한다고 알려져있다.  
이 엔진은 역시 interpreter이고 성능을 위해 JIT 컴파일 방식을 사용한다고 한다. JIT이란 자주 쓰는 코드는 기계어로 번역한다는건데 실행 직후엔 인터프리터와 속도가 같으나 기계어 번역이 많아질수록 속도가 인터프리터보다 압도적으로 빨라진다고 한다. 추가적으로 싱글 스레드 동작이지만 멀티 스레드만큼 빠른 비법 등도 찾으면 나오지만, 일단 현재 단계에서 정리할 필요가 없어서 넘어가도록 하겠다.

# 3. JS와 DOM
Node를 좀 제외하고 DOM 기준으로 JS를 정리하고자 한다. 일반적으로 HTML 코드에 style이 입혀지고 그 뒤에 script Tag로 JS가 HTML과 CSS에 접근하거나 서버와 통신하는 역할을 진행한다. 여기서 웹은 script Tag를 만나면 어떻게 할까? TAG를 만날 때마다 웹 엔진이 Element들을 가지고 파싱하여 DOM Tree에 추가한다. 그리고 계산된 CSSOM을 가지고 와서 DOM Tree를 수정한다. 이 과정을 통해서 우리는 브라우저에 있는 완성된 페이지를 볼 수 있다. script Tag가 HTML Tag지만 웹 엔진이 파싱하다가 script tag를 만난다면 일반적인 엘리먼트와는 다르게 동작한다. 바로 해당 script를 수행할 수 있도록 브라우저가 가지고 있는 모든 자원을 할당해 준다(Tab 하나당 1 Thread라고 보면 된다고 한다).  
script Tag를 읽은 후 동작되는 Java Script 코드는 데이터를 수정하든, HTML 태그에 대한 CRUD를 진행하든, CSS를 건들든 무조건 브라우저 위에서 작동한다. DOM에 접근하려면 Document나 window같은 브라우저에서 제공해주는 객체를 통해 접근하여 동작한다. 이러한 규칙 때문에 JS를 사용할때에 DOM 초기화 타이밍을 잘 파악해야 한다. 만약 서버에서 받아오는 일을 HTML 코드 중간에 넣었을 때 서버에서 응답을 안 줘 버리면 예외 사항에 대처조차 못하는 웹 페이지가 되는 것이다.
